# Contributing to Sapri Projects 🚀

First off, thank you for considering contributing to the Sapri ecosystem (**Sapri Aurea** & **sapriSort**). It’s people like you who make these tools better for everyone.

Below are the guidelines to help you get started.

---

## 🧠 Our Vision
We aim to bridge the gap between **pure mathematics** (Number Theory, Golden Ratio, Quantum Logic) and **high-performance computing**. We value:
* **Efficiency:** Code that respects CPU/GPU cycles.
* **Clarity:** Mathematical logic that is documented and readable.
* **Innovation:** Unconventional approaches to classic problems (like our Vectorial Lucas-Lehmer Test).

---

## 🛠️ How Can You Contribute?

### 1. Reporting Bugs
* Check the **Issues** tab to see if the bug has already been reported.
* If not, open a new issue. Include your OS, language version (C, Python, Node.js), and a small snippet to reproduce the error.

### 2. Suggesting Enhancements
We love new ideas! Whether it's a new sorting algorithm for `sapriSort` or a new mathematical constant implementation for `Sapri Aurea`:
* Open an issue labeled `enhancement`.
* Explain the mathematical basis and the expected performance gain.

### 3. Code Contributions (Pull Requests)
1. **Fork** the repository.
2. **Clone** your fork: `git clone git@github.com:YourUsername/sapriSort.git`
3. **Create a branch**: `git checkout -b feature/amazing-new-algorithm`
4. **Commit your changes**: Use clear, descriptive messages.
5. **Push to the branch**: `git push origin feature/amazing-new-algorithm`
6. **Open a Pull Request**.

---

## 📝 Coding Standards
* **Documentation:** Every new algorithm must include a brief explanation of its complexity ($O(n)$) and mathematical logic.
* **Clean Code:** Follow the existing style of the repository.
* **Testing:** If you add a new sort, ensure it passes the verification tests for both small and large datasets.

---

## 💎 Specific Areas of Interest
We are currently looking for help with:
* **GPU Acceleration:** Porting our NumPy/Vectorial logic to CuPy or PyCUDA.
* **Advanced Sorting:** Implementations of Bitonic Sort or Radix Sort optimized for modern architectures.
* **Math Visualization:** Tools to visualize the Lucas-Lehmer sequence or Golden Ratio distributions.
